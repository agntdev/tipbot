# fix-8b820399897dfe8e — bot.catch() overwrites toolkit's error handler — onError logger never fires

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TIP

`toolkit/bot.ts:47-50` installs a `bot.catch()` that calls `opts.onError` (the `logger.error` callback passed at `src/bot.ts:33`). Then `src/bot.ts:119-128` calls `bot.catch()` AGAIN, which in grammY REPLACES the previous handler. The second handler silently swallows errors (no logging at all) and only conditionally replies to the user. Result: ALL middleware errors are silently lost — the `onError` logger is never invoked, and the second handler doesn't call it either. Sensitive production errors are invisible.

The fix: the second catch handler should call `opts.onError` first (or be merged into `createBot`'s catch), not overwrite it.
