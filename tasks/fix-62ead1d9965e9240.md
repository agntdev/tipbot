# fix-62ead1d9965e9240 — Key name mismatch: tips_served vs tip_count breaks E1T3 initialization

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TIP

E1T3 spec: "ensure 'tips_served' key exists with a default value of 0". `src/persistent.ts:69` inserts `tips_served=0` as required. But the `/tip` handler (`src/bot.ts:78`) calls `store.incr("tip_count")` and the `/count` handler (`src/bot.ts:85`) calls `store.get("tip_count")`. The key `tips_served` is never read or written by any command — the initialization is dead code. The commands happen to work because they auto-create `tip_count` lazily on first use (`incr` uses `INSERT OR IGNORE`), but the intended pre-seeded counter from E1T3 is entirely disconnected from what the commands actually operate on.
