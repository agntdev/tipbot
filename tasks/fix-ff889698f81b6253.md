# fix-ff889698f81b6253 — Unhandled promise rejection from un-awaited setMyCommands can crash the process

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TIP

`bot.api.setMyCommands(...)` at `src/bot.ts:36` returns a Promise that is never awaited nor chained with `.catch()`. If the Telegram API call rejects (network error, invalid token, rate limit), this becomes an unhandled promise rejection. In Node.js ≥15, unhandled rejections terminate the process by default. This call executes before the bot starts polling, so a failure at startup crashes the bot with no recovery.

The fix: either `await` the call (and wrap in try/catch), or `.catch()` the promise to log the error without crashing.
